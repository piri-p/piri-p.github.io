### Think about what to store in stack

For example,
* Waitlist of items that don't have the answer yet (can be monotonic stack like [1475. Final Prices With a Special Discount in a Shop](https://leetcode.com/problems/final-prices-with-a-special-discount-in-a-shop/description/))
  * Need to know index to backfill something or not? (e.g. 1475 - yes because during pop operations the original index was lost)
* It pops both items or only single ones, and stopping criteria.
* Can sometimes stores just number of occurences rather than actual characters.
* Exit condition (e.g. stack is empty? need to handle loop exit one more time? pop last item and return - like in Reversed Polish Notion?)

### Handle digit encounter

`num = 10 * num + int(digit)`
